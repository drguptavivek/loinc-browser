# LOINC MCP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a Go-native LOINC MCP server with stdio and HTTP transports, live Markdown-backed agent context, and a repository skill for agents.

**Architecture:** Add `internal/mcpserver` as the shared MCP implementation over `loinc.Store`. Wire it into `loinc-browser mcp` for stdio and `loinc-browser serve --mcp` for HTTP, with docs loaded from `docs/agent` at request time.

**Tech Stack:** Go 1.26, `github.com/modelcontextprotocol/go-sdk/mcp`, existing `loinc.Store`, existing net/http server, Svelte checks unchanged.
