# LOINC v1 EMR Form-Builder API Execution Plan

Goal: build a normalized `/api/v1` LOINC API for EMR form builders with ranked term search, hierarchy browsing, panels, answer lists, parts, groups, source/copyright metadata, pagination, and HATEOAS links.

Execution rules:
- Do not revert unrelated worktree changes.
- Do not create compatibility tables.
- Do not query `map_to`, `term_accessories`, `hierarchy_nodes`, `hierarchy_nav_edges`, or `hierarchy_term_members`.
- Start by replacing the partial normalized relationship test failure around `HierarchyNode.NodeID`.
- Run `go test ./...` after Go tasks and `npm --prefix web run check && npm --prefix web run build` after frontend tasks.

Task order:
1. Add v1 response types, links, pagination, ranking defaults, and shared term-list params.
2. Replace search with one normalized term-list query path: default `ACTIVE`, `status=*`, usage filters, rank modes, pagination, and HATEOAS links.
3. Add `/api/v1` server routes for health, terms search/top/detail/fit, hierarchy, relationships, source metadata, and normalized resource browsing.
4. Rewrite term relationships from normalized tables: `loinc_map_to`, `loinc_part_links`, `loinc_answer_list_links`, `panel_items`, `group_loinc_terms`, and hierarchy occurrence/subtree tables.
5. Implement hierarchy browsing on `hierarchy_occurrences.node_id`.
6. Add panel, answer-list, part, group, and source/copyright focused endpoints.
7. Add route-oriented SQLite indexes without reintroducing JSON/EAV tables.
8. Update OpenAPI and docs for v1 only.
9. Migrate Svelte client calls to `/api/v1` and stable hierarchy `nodeId` keys.
10. Final verification: `go test ./...`, `npm --prefix web run check`, `npm --prefix web run build`, and API smoke checks against `./data/loinc-normalized.sqlite` when available.
